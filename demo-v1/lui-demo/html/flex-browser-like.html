Flex browser-parity follow-up

This page isolates the two flex behaviors improved in the previous pass: column-direction wrapping with an indefinite main size, and cross-axis percentage sizes that should stay unresolved when the container cross size is indefinite.

1. Column wrap + no explicit height

Browser behavior keeps this as a single column because the main axis height is indefinite. The renderer should not wrap these boxes into extra columns just because flex-wrap: wrap is set.

In browsers, height: 50% on a flex item does not resolve against an indefinite container height. That means it should behave like auto here, allowing align-items: stretch to stretch all items to the line height.
